### **ZetaChain 上发生的过程**

**1\. 跨链消息传递流程**

**在 Sepolia 上：**

solidity

```
// 用户调用 CrossChainSwap 合约
function swapETHForZETA(uint256 minAmountOut) external payable {
    // ETH 被发送到 ZetaConnector
    zetaConnector.send{value: msg.value}(
        abi.encode(
            msg.sender,        // 用户地址
            address(0),        // 接收代币（0表示ZETA）
            msg.value,         // 发送金额
            minAmountOut       // 最小输出
        )
    );
}
```

**在 ZetaChain 上：**

solidity

```
// ZetaChain 的 UniversalApp 处理跨链调用
function onCall(
    MessageContext calldata context,
    address zrc20,
    uint256 amount,
    bytes calldata message
) external override {
    // 1. 验证调用权限
    require(msg.sender == address(connector), "Invalid caller");
    
    // 2. 解析跨链消息
    (address user, address targetZRC20, , uint256 minOutput) = 
        abi.decode(message, (address, address, uint256, uint256));
    
    // 3. 计算ZETA输出数量
    uint256 zetaAmount = amount * exchangeRate;
    
    // 4. 向用户地址发送ZETA
```

**2\. 具体处理步骤**

**步骤 1 - 资产锁定**

-   我的 0.01 ETH 在 Sepolia 上被锁定
    
-   交易通过 ZetaConnector 发送到 ZetaChain
    

**步骤 2 - 跨链验证**

-   ZetaChain 的 Observer 节点监控 Sepolia 链
    
-   验证交易的有效性和最终性
    
-   TSS（阈值签名方案）确认资产锁定
    

**步骤 3 - ZRC20 铸造**

-   ZetaChain 上铸造等值的 ETH.ZRC20 代币
    
-   这些 ZRC20 代表锁定的 Sepolia ETH
    

**步骤 4 - Swap 执行**

-   在 ZetaChain 上执行 ETH.ZRC20 → ZETA 交换
    
-   使用内置的流动性池计算汇率
    
-   确保输出满足最小数量要求
    

**步骤 5 - 结果分发**

-   ZETA 代币发送到我的 ZetaChain 地址
    
-   交易状态更新，跨链完成
    

**3\. 架构中的位置**

根据 ZetaChain 架构文档，这个过程涉及：

1.  **源链组件 (Sepolia)**:
    
    -   `ZetaConnector`: 跨链消息入口点
        
    -   `CrossChainSwap`: 用户交互合约
        
2.  **ZetaChain 核心**:
    
    -   **Observer Network**: 监控各链事件
        
    -   **TSS Module**: 管理跨链资产安全
        
    -   **Cross-Chain Message Router**: 路由和处理消息
        
    -   **ZRC20 System**: 跨链代币管理
        
3.  **目标链组件 (ZetaChain)**:
    
    -   `UniversalApp`: 业务逻辑处理
        
    -   `ZETA Token`: 原生代币分发

### _ZRC-20 与普通 ERC-20 的直观区别_

| 特性维度 | 普通 ERC-20 | ZRC-20（ZetaChain 实现） |
| --- | --- | --- |
| 原生跨链能力 | 无。部署在单一链上（如以太坊主网），跨链需借助第三方桥或跨链协议。 | 内置原生跨链。一个 ZRC-20 资产可同时在多条连接的链上存在并转移，由 ZetaChain 底层协议保证跨链一致性。 |
| 资产形态 | 单一链上合约实例。不同链上的“同名”资产实质是不同的合约，需通过映射建立联系。 | 统一跨链资产。在 ZetaChain 上表现为一个通用资产，在其他链上可映射为当地标准的代币（如 ERC-20、BEP-20）。 |
| 跨链交互方式 | 需要调用跨链桥合约，进行锁定/铸造或销毁/解锁，流程复杂，安全性依赖桥。 | 通过 ZetaChain 的 deposit / withdraw 方法，直接在 ZetaChain 上操作跨链转账，流程标准化，由 ZetaChain 节点网络保障安全。 |
|   |   |   |
| Gas 与手续费 | Gas 以所在链原生代币支付（如 ETH）。跨链时需在源链与目标链分别支付 Gas，并可能支付桥接费用。 | 在 ZetaChain 上操作时使用 ZETA 作为 Gas。跨链交易的手续费统一用 ZETA 结算，简化用户体验。 |
| 安全性假设 | 依赖所在链的安全性。跨链时额外依赖桥接合约或第三方验证者的安全性。 | 依赖 ZetaChain 网络（基于 Tendermint + POS）的安全性，以及连接链的轻节点验证。 |
|   |   |   |
| 适用场景 | 单链 DeFi、单链通证、需跨链时作为源链或目标链的本地资产。 | 多链统一资产、跨链 DeFi、跨链游戏资产、通用会员通行证等。 |

### _「通用资产」的应用场景举例_

1\. 跨链储蓄与收益聚合

**场景描述**：用户持有多种链上的闲置资产（如 ETH 在以太坊上，BNB 在 BSC 上，MATIC 在 Polygon 上），希望统一存入一个协议中获取收益。

-   **传统方式**：用户需要在每条链上分别寻找当地的收益协议，分别操作，资金分散，管理复杂。
    
-   **使用 ZRC-20 通用资产**：
    
    -   用户将各链资产通过 `deposit` 转为对应的 ZRC-20 通用资产（如 ETH → zETH，BNB → zBNB）。
        
    -   所有 zAssets 统一进入 ZetaChain 上的一个收益聚合器。
        
    -   聚合器根据策略将资产跨链调配至高收益协议（如将 zETH 部分跨链至 Avalanche 上的 Benqi 借贷，部分留在 ZetaChain 上做 LP）。
        
    -   用户只需与 ZetaChain 上一个合约交互，即可实现多链资产的统一管理和收益最大化。
        

2\. 通用 NFT 通行证（Universal Membership Pass）

**场景描述**：一个游戏或社交平台发行会员 NFT，持有者可在多条链上的合作 DApp 中享受权益。

-   **传统方式**：在不同链上分别发行相似的 NFT，用户需要在不同链上分别持有，权益不互通，体验割裂。
    
-   **使用 ZRC-20 通用资产**：
    
    -   平台在 ZetaChain 上发行一个通用 NFT（作为 ZRC-721 或类似跨链标准资产）。
        
    -   用户购买后，该 NFT 在 ZetaChain 上存在，但可通过 ZetaChain 的跨链消息系统，让以太坊、Polygon、BSC 上的合作 DApp 都验证到用户的持有状态。
        
    -   例如：
        
        -   在以太坊上的一个 DeFi 协议中，NFT 持有者可获得更高存款利率。
            
        -   在 Polygon 上的一个游戏中，NFT 可解锁特殊角色或道具。
            
        -   在 BSC 上的一个商城 DApp 中，NFT 可享受购物折扣。
            
    -   用户只需持有这一个通用 NFT，即可在所有连接的链上享受权益，实现真正的“一证通”。

1\. Universal App 是什么？

**Universal App** 是一种构建在 ZetaChain 上的去中心化应用。它的核心特点是“通用性”和“无缝连接”。

-   **跨链无需桥接**：传统的 DApp 如果要在多条链上运行，需要在每条链上都部署合约，然后用复杂的“桥”来让它们通信和转移资产。Universal App 只需要在 ZetaChain 上部署一次，就能直接管理和与所有连接的链（比如 Bitcoin, Ethereum, Solana）进行交互。
    
-   **可管理外部资产和状态**：它不仅能处理 ZetaChain 上的资产，还能直接读取和修改其他链上的状态。最关键的是，它甚至可以管理那些**没有智能合约功能**的链（如 Bitcoin, Dogecoin）上的原生资产，就像在操作一条智能合约链一样。
    
-   **用户体验统一**：用户可以从任何一条链发起交易，与这个应用交互，而无需关心背后复杂的跨链逻辑。应用逻辑全部由 ZetaChain 上的智能合约统一处理。
    

**简单比喻**：就像是一个“万能遥控器”（Universal App），你只需要拿着它（在 ZetaChain 上），就能直接控制家里所有不同品牌、不同协议的电器（各条区块链），而不需要为每个电器单独找一个适配器（跨链桥）。

2\. Gateway 大概做什么？

**Gateway** 是 ZetaChain 生态系统中的一个关键组件，它扮演着“入口”或“适配器”的角色。

-   **主要功能是资产出入**：它的核心工作是帮助用户将其他链上的原生资产（如 ETH, BTC, SOL）“存入” ZetaChain，并在需要时将这些资产“取回”到原链。
    
-   **封装与解封装**：
    
    -   **存入时**：当用户将 ETH 从 Ethereum 链发送到 Gateway 合约后，Gateway 会通知 ZetaChain，随后在 ZetaChain 上为用户铸造出等量的、可互操作的“封装资产”（如 `ZETA-ETH`）。
        
    -   **取出时**：当用户在 ZetaChain 上销毁 `ZETA-ETH` 时，ZetaChain 会通知 Ethereum 上的 Gateway 合约，从而将原生的 ETH 释放给用户。
        
-   **交互的起点**：对于用户来说，与一个 Universal App 交互，第一步往往就是通过 Gateway 将资产跨到 ZetaChain 上。
    

**简单比喻**：Gateway 就像一个**银行的柜台**。你想把美元（Ethereum 上的 ETH）换成旅行支票（ZetaChain 上的 `ZETA-ETH`）以便在全球通用，你需要去柜台办理。当你旅行结束，想换回美元时，你同样需要去柜台把旅行支票换回来。

​_RPC&Faucet&Explorer_

核心概念

1.测试网：一个功能与主网完全相同的区块链网络，但使用没有真实价值的代币。用于开发、测试和部署智能合约，而无需承担财务风险

2.RPC：远程过程调用端点。你的应用程序（如钱包、DApp前端）通过RPC URL与区块链网络进行通信（查询数据、发送交易）

3.Faucet：“水龙头”。一个免费发放测试网代币的服务，让你可以支付测试网上的交易Gas费

4.Explorer：“区块链浏览器”。一个网站，用于查看区块、交易、地址、合约等所有；链上活动的详细信息
