# Omni-Savings —— 全链跨链储蓄金库（跨链收益聚合）

### 目标用户

-   想在多链赚收益但不想频繁切链的普通用户
    
-   想用 BTC / ETH / BNB / USDT 做 DeFi 的用户
    
-   想统一管理多链资产、但不愿意维护多条链账户的“轻量玩家”
    

### 想解决的问题

**当下 DeFi 的核心痛点：**

-   用户资产分散在不同链（ETH / Polygon / BSC / Arbitrum / BTC）
    
-   想赚钱必须：
    
    -   在每条链桥资产
        
    -   更换钱包网络
        
    -   找对应链的 Yield、Farm、Pool
        
    -   维护多条链的复杂仓位
        
-   成本高、体验差、普通人完全不会用
    

**Omni-Savings 的价值：**

> “用户把资产从任何链充值进来，  
> 合约自动统一处理收益，并且可以跨链取出。”

让用户体验变成：

-   从 **任意链** 存资产（原链不动）
    
-   收益在 ZetaChain 统一产生
    
-   想提现时 → 到 **任意链地址**
    

### 通用资产（ZRC-20）+ 跨链方式

**核心机制：**

1.  用户从 ETH / BSC / Polygon / BTC 存资产  
    → 在 ZetaChain 变成对应的 ZRC-20
    
    -   ETH → ETH.ZRC20
        
    -   BTC → BTC.ZRC20
        
    -   USDT (Polygon) → USDT.POLY.ZRC20
        
2.  Vault 合约统一处理这些 ZRC-20
    
    -   做收益策略（例如统一借贷、LP、restake 等）
        
    -   所有链的资产变成统一“格式”，不需要多合约多链版本
        
3.  用户提现时：
    
    -   Omni-Savings 查询用户的所有份额
        
    -   ZetaChain 使用跨链 Messaging / TSS  
        → 把资产自动“送回”用户想要的链

# 2025-11-30
<!-- DAILY_CHECKIN_2025-11-30_START -->

我是在Sepolia测试网通过官方 Sender 合约发起调用的。

具体流程是：

1.  Sender 合约运行在 EVM 链。
    
2.  我用 Hardhat 运行脚本 `send-message.js` 发起消息：
    
    ```
    npx hardhat run scripts/send-message.js --network sepolia
    ```
    
3.  这笔交易由 Sepolia 打包成功，并通过 ZetaChain的跨链消息系统发送到 ZetaChain。
    

调用本质上是：“从非 ZetaChain 链 → 发消息 → 到 ZetaChain”。

* * *

当消息从 EVM 链传入后：

1.  ZetaChain 的观察者节点监控到 EVM 链上的事件
    
2.  将这条跨链消息提交到 ZetaChain 的Universal EVM
    
3.  我部署在 Athens-3 上的 `ZetaReceiver` 合约收到消息
    
4.  合约根据消息内容触发 `MessageReceived` 事件，链上记录如下：
    

```
Hello ZetaChain from Sepolia!
```

你可以在 ZetaChain Explorer 中看到：

-   来自 EVM 的跨链消息
    
-   消息数据
    
-   Receiver 合约的执行日志
    
-   ZETA Gas 消耗情况
    

**跨链消息成功地从 Sepolia → ZetaChain 完成一次执行。**

# 2025-11-29
<!-- DAILY_CHECKIN_2025-11-29_START -->


ZRC-20 是 ZetaChain 对多链资产的统一抽象，它不仅是代币标准，更是跨链资产交互的核心基础。开发者使用 ZRC-20 可以构建通用应用（Universal Apps），从而实现“用一次代码，操控所有链上的资产”。相对的，ERC-20 仅在单个链有效，没有跨链能力。

一个通用资产的典型场景是：跨链储蓄 Vault。用户从 ETH/BSC/Polygon/BTC 等链存入资产，ZetaChain 在内部将它们统一转换成 ZRC-20，由一个合约管理并产生收益。用户以后可以选择领取到任何链，开发者不需要维护多条链的代码。
