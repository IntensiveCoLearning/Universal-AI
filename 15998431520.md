### 自己想做的第一个 Universal App 想实现的“打印 / 记录 / 简单逻辑”是什么

它是一个记录瞬间感悟的 UApp 底部有两个 tab，发布 & 浏览,发布部分有一个输入框，一个上传文件的按钮，用户可以把当时的感受记录下来，也可上传那个瞬间的照片，点击提交按钮，就会被算法编译成固定长度的哈希值，利用大模型的能力，根据用户上传的信息，生成一张图片，直观形象的表现出用户本次上传的内容，把 hash 值融合到图片底部，标记上时间日期，上传到区块链，生成一个 NFT，存入用户账户中。像一个记忆方块被永久的存贮在去中心化网络上，永远不会被篡改，也不会消失，也没有人知道是谁的记忆方块，在 Uapp 的另一个浏览模块，用户可以浏览其他人的记忆方块，上下滑动可查看由大模型生成的 NFT 封面，如果感兴趣可以点击打开查看背后的故事，可以点赞和评论，优秀的记忆方块发布者，会得到一定量的代币奖励。

-   使用 CLI + Hardhat / Foundry？
    
-   用本地链还是测试网？

### Universal App 是什么？

标准含义是 - 通用应用，比如在苹果生态中，一个 App 可以在 iPhone 上运行，并且可以在 iPad 或者 Mac 中正常使用，不会因为运行端的不同而无法运行，那么这个 App 就是 Universal App 。

在 web3 中则代表，用户可专注于 Dapp 的功能，而无需关心繁琐的网络切换，登录状态切换，区块链的切换，符合这样需求的 Dapp, 就可以称之为 Universal App 。

### Gateway 大概做什么？

标准含义是 - 网关，比如局域网和互联网的链接，路由器就是 Gateway 。

在 web3 中则代表，连接去中心化底层并提供服务给上层调用，使开发者能像调用普通 API 一样，轻松使用去中心化网络。

### 画一张简单的架构图：

> ZetaChain 中心 + Bitcoin / Ethereum / Solana 等外围链 + Gateway。

ZetaChain 是一个独立的 L1 区块链，像一个枢纽聚合了 Ethereum、Bitcoin 等核心链，通过 Getaway 作为出入口向开发者提供便利的与去中心化网络之间的交互。
