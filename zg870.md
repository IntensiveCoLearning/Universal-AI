3.  在 ZetaChain 链上执行实际的 Swap 操作。
    
4.  将目标 token 发送到我对应的 ZetaChain 地址，并产生最终交易记录

**一、ZRC-20 和普通 ERC-20 的直观区别**

1.  **所在链不同**
    
    -   **ERC-20**：部署在某一条链上，例如 Ethereum、BSC 等，只能在该链内部使用。
        
    -   **ZRC-20**：部署在 ZetaChain 上，天然支持多链。一个 ZRC-20 代币可以在多条链之间流动，而不用做 “包装” 或跨链桥。
        
2.  **转账范围不同**
    
    -   **ERC-20**：`transfer` 就是链内转账，不能自动跨链。
        
    -   **ZRC-20**：可以直接跨链转账，ZetaChain 会负责把结果同步到目标链。
        
3.  **资产模型不同**
    
    -   **ERC-20**：每条链的代币都是分开的版本，要跨链需要桥。
        
    -   **ZRC-20**：只有一个统一的资产，不管在哪条链使用，它代表的都是同一个余额。
        
4.  **安全方式不同**
    
    -   **ERC-20** 跨链主要靠第三方桥，风险大。
        
    -   **ZRC-20** 的跨链由 ZetaChain 自己完成，不需要依赖外部桥。
        

* * *

## **二、通用资产的应用场景示例**

### **例子：跨链储蓄账户**

1.  用户可以在任意链（比如 ETH、Polygon、BSC）存入同一种 ZRC-20 资产。
    
2.  存进去的资产会进入 ZetaChain 统一管理，无论从哪条链存入，都加到同一个账户余额。
    
3.  用户也可以在任意链取出，不需要跨链桥，也不需要换成包装代币。
    
4.  效果就像一个“全链通用”的余额账户。

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract Counter {
    uint256 public count;

    event CounterUpdated(uint256 newCount);

    // 增加计数器
    function increment() public {
        count += 1;
        emit CounterUpdated(count);
    }

    // 减少计数器
    function decrement() public {
        require(count > 0, "Counter: cannot go below zero");
        count -= 1;
        emit CounterUpdated(count);
    }

    // 设置计数器为指定值
    function setCount(uint256 _count) public {
        count = _count;
        emit CounterUpdated(count);
    }
}
```
