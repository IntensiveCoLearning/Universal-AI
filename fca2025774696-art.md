# Universal Apps（通用应用）

## 核心概念

Universal App 是部署在 ZetaChain 上的智能合约，其核心特点是具备原生的全链互操作性。这意味着一个单一的合约可以直接与所有连接的区块链（如 Ethereum、BNB Chain、Bitcoin 等）进行交互。

## **主要功能与能力**

1\. 接收信息与资产：Universal App 可以从任何连接的链上接收合约调用、消息和代币转账。用户通过与目标链上的一个“网关合约”交互来触发这些操作。

2\. 发送信息与资产：它也可以主动在其他链上触发合约调用和代币转账。

3\. 简化跨链交易：通过将多个步骤（如接收资产、跨链兑换、发送资产）整合在一个合约调用中，Universal App 能够将复杂的跨链交易简化为用户只需签署一次的简单操作。

## **关键技术特点**

EVM 兼容：基于 ZetaChain 的“通用 EVM”构建，现有以太坊合约可以轻松迁移，并通过少量修改获得全链能力。

ZRC-20 标准：来自不同链的原生 gas 代币和主流资产（如 ETH、BNB）在 ZetaChain 上被统一表示为 ZRC-20 代币。这些代币可以无许可地撤回至其原始链。

Gas 抽象：系统自动处理跨链交易所需的目标链 gas 费用。用户无需持有目标链的代币来支付手续费，费用会从他们发送的代币中自动扣除和兑换。

比特币兼容：Universal App 同样支持比特币网络，比特币用户可以通过向特定地址发送交易来与合约交互。

## **总结**

总而言之，Universal App 是 ZetaChain 提供的强大工具，旨在让开发者能够构建真正的全链去中心化应用，使用户无需关心底层多条区块链的复杂性，即可实现无缝的跨链交互

# Gateway（网关）

### **_Gateway 是连接外部区块链（如 Ethereum、Bitcoin）与 ZetaChain 上 Universal Apps（通用应用） 的统一入口和出口。_**

## 核心角色

Gateway 充当一个桥梁，所有跨链交互都必须通过它。它在两个方向上发挥作用：

1\. 入站交易：处理从外部链到 ZetaChain 的调用。

2\. 出站交易：处理从 ZetaChain 到外部链的调用。

## Gateway 的实现形式

Gateway 的具体实现取决于它所在的链：

EVM 链（如 Ethereum, BNB Chain）：一个网关智能合约。

Solana：一个网关程序。

Bitcoin：一个由验证者网络管理的门限签名方案（TSS）地址。每个连接的区块链上只有一个网关，所有与 ZetaChain 的通用应用的交互都通过这个唯一的网关进行。

## 主要功能

### 1\. 入站功能（从外部链到 ZetaChain）

网关支持用户和合约从外部链向 ZetaChain 发送资产和指令，包括：

存入原生代币（如 ETH、BTC）或支持的代币（如 ERC-20）。

在存入资产的同时，向一个通用应用发送合约调用（附带任意数据）。

仅发送数据调用（不附带资产）。

### 2\. 出站功能（从 ZetaChain 到外部链）

网关支持通用应用向外部链发送资产和指令，包括：

将 ZRC-20 代币提现为对应链的原生代币或 ERC-20 代币。

在提现代币的同时，在目标链上触发一个合约调用。

仅触发合约调用（不转移资产）。

### 其他重要特性

当前限制：目前一次操作只能存入或提现一种资产，未来将支持多资产操作。

错误处理与退款：如果跨链调用在目标链上失败，网关支持灵活的退款机制。资金可以被退回到源链上的指定合约，或者直接退回到一个普通用户地址（EOA）。

## 总结

总而言之，Gateway 是 ZetaChain 实现全链互操作性的关键基础设施。它通过标准化的接口，简化了用户和开发者与不同区块链交互的复杂性，使得跨链交易像单链交易一样简单可靠。

# 一 CLI

**工具定位：**一个统一的命令行工具，用于简化 ZetaChain 生态系统的开发流程。

**核心优势：**无缝的多链交互。开发者可以使用这一个工具直接与 EVM、Solana、Bitcoin、Sui 和 TON 等多种区块链进行交互，无需切换不同的链特定工具。

**主要功能：**项目脚手架、本地开发环境管理、账户管理、区块链数据查询、执行跨链交易等。

## 主要内容详细分类

**1\. 核心特性与能力**

-   从模板搭建新项目：快速创建新的通用应用程序项目结构。
    
-   一键启动本地开发网络：通过单个命令启动一个包含多链（EVM, Solana 等）的本地开发环境。
    
-   丰富的查询功能：查询跨链手续费、合约信息、各个链上的余额、跨链交易状态、支持的代币等。
    
-   执行跨链操作：直接通过命令行在不同链（如 Solana, Bitcoin）和 ZetaChain 上的通用应用之间进行跨链调用和资产转移。
    

**2\. 安装与快速开始**

系统要求：Node.js ≥ 18, Git, 以及可选的 Docker（用于运行本地网络）。

安装方式：

免安装运行：npx zetachain@next new

全局安装：npm install -g zetachain@latest

常用命令示例：

zetachain new：创建新项目。

zetachain localnet start：启动本地网络。

zetachain query balances：查询跨链余额。

**3\. 核心命令模块详解**

文档详细列出了所有命令和选项，主要可分为以下几大模块：

a. 账户管理

zetachain accounts create/import/list/show/delete

支持为不同的链类型创建和管理账户，包括 EVM, Solana, Sui, Bitcoin, TON。

b. 查询功能

zetachain query balances：查询指定地址在不同链上的原生代币和 ZETA 余额。

zetachain query cctx：根据交易哈希实时查询跨链交易的状态。

zetachain query fees：查询全链和跨链消息的手续费。

zetachain query tokens：列出和查看 ZRC-20 代币的详细信息。

zetachain query chains：列出所有支持的链及其详细信息。

c. 跨链交易操作（核心功能）

这是 CLI 最强大的部分，为每条主要链提供了专门的命令集：

从 ZetaChain 发起：zetachain zetachain call/withdraw/withdraw-and-call。从 ZetaChain 调用或提现资产到连接的链上。

从 EVM 链发起：zetachain evm call/deposit/deposit-and-call。从 EVM 链存款或调用 ZetaChain 上的合约。

从 Solana 发起：zetachain solana call/deposit/deposit-and-call。

从 Sui 发起：zetachain sui deposit/deposit-and-call。

从 TON 发起：zetachain ton deposit/deposit-and-call。

从 Bitcoin 发起：支持两种方式：

铭文：zetachain bitcoin inscription call/deposit...

备忘录：zetachain bitcoin memo call/deposit...

d. 本地开发网络

zetachain localnet start/stop/check：管理本地开发环境，可以指定要启动的链。

e. 其他实用命令

# **ZetaChain的总结**：

**一句话总结：**ZetaChain 旨在成为一个通用的、能够原生连接所有区块链的底层平台，让开发者能够像开发单链应用一样，轻松构建跨链应用（全链应用）。

其终极目标是解决当前区块链生态的碎片化问题，成为去中心化世界的“通用入口”和“连接器”。

## **一、核心问题：为什么需要 ZetaChain？**

当前区块链生态存在以下痛点，而 ZetaChain 正是为了解决它们而生：

痛点 ZetaChain 的解决方案

孤岛效应：每条链都是一个封闭生态，资产和数据难以互通。 枢纽模型：ZetaChain 自身作为中心枢纽，统一连接和管理所有链。

开发复杂：构建跨链应用需要在多条链上部署合约，并维护复杂的跨链通信逻辑。 通用应用：开发者只需在 ZetaChain 上部署一次智能合约，即可管理所有连接链的交互。

用户体验差：用户需要手动使用跨链桥，操作繁琐，成本高，风险大。 无缝体验：用户在一个链上的单一操作，即可由通用应用自动编排复杂的多链交易。

资产形式不统一：跨链后产生大量的封装资产，流动性分散，安全性依赖第三方。 原生资产转移：通过 ZRC-20 标准，实现资产在不同链间的原生转移，无需封装。

## **二、核心技术架构与组件总结**

ZetaChain 通过一套精密的组件协同工作，实现上述愿景。

-   **通用应用**
    

\- 定义：部署在 ZetaChain 上的智能合约，但其能力不局限于 ZetaChain 本身。

\- 核心能力：

\- 读取任何连接链的状态。

\- 向任何连接链发送资产和触发合约调用。

\- 编排涉及多条链的复杂交易流程。

-   **关键技术支柱**
    

1.  通用权益证明：基于 Cosmos SDK，提供网络安全和快速最终性。
    
2.  通用 EVM：完全兼容以太坊 EVM，开发者可使用 Solidity 和熟悉的所有工具。
    
3.  Omnichain 智能合约：具备原生跨链能力的合约的执行环境。
    

-   **关键协议与组件**
    

\- CCTX：跨链交易的全局记录。每一个跨链操作都会生成一个 CCTX 来追踪其状态（待处理、已确认、已回退等），是保证操作可追溯性的核心。

\- Gateway（网关）：用户与 ZetaChain 交互的统一入口。无论是存入资产还是调用合约，用户都通过连接链上的网关合约进行。

\- 入站：从外部链存入资产或调用 ZetaChain 合约。

\- 出站：从 ZetaChain 提取资产或调用外部链合约。

\- ZRC-20：ZetaChain 上的代币标准，用于代表所有连接链上的原生资产（如 ETH 在 ZetaChain 上就是 ZRC-20 形式的代币）。它是实现资产跨链的桥梁。

-   **Gas 费机制**
    

\- 在 ZetaChain 上执行：用户从连接链调用通用应用时，在 ZetaChain 端的合约执行是 Gasless（无Gas费） 的，用户只需支付源链的 Gas。

\- 从 ZetaChain 提款/调用：通用应用需要支付目标链的 Gas，这笔费用以目标链 Gas Token 的 ZRC-20 形式支付。

# ZetaChain 本地开发环境的全过程

## 第一阶段：安装核心前置依赖

**1\. 安装 Node.js 和 npm**

目的：这是运行 ZetaChain CLI 和大多数 JavaScript/TypeScript 工具链的基础。

操作：

访问 Node.js 官网。

强烈建议下载并安装 v21.x 或更高版本的 LTS（长期支持）版本。这将同时安装 Node.js 和其包管理器 npm。

打开终端（Terminal 或 Command Prompt），运行以下命令验证安装是否成功：

node --version # 应输出 v21.x.x 或更高 npm --version # 应输出对应的 npm 版本号

**2\. 安装 Git**

目的：用于克隆模板项目和进行版本控制。

操作：

访问 Git 官网下载并安装。

安装后，在终端中验证：

git --version

**3\. 安装 jq**

目的：一个轻量级的命令行 JSON 处理器，用于解析脚本输出（尤其在 Localnet 中）。

操作：

macOS： 使用 Homebrew 安装是最简单的方式。

brew install jq

Linux (Ubuntu/Debian)：

sudo apt-get update && sudo apt-get install jq

Windows： 建议通过 WSL 安装 Linux 发行版，然后在 WSL 内使用上述 Linux 命令安装。或者，可以从 官网 下载 exe 文件并配置环境变量。

验证：

jq --version

**4\. 安装 Foundry**

目的：一个快速的以太坊开发工具包。ZetaChain 的 Localnet 依赖于它来运行本地 EVM 环境。

操作：

打开终端，运行以下命令（适用于 macOS 和 Linux）：

curl -L [https://foundry.paradigm.xyz](https://foundry.paradigm.xyz) | bash

安装脚本会输出提示。完成后，重启你的终端或运行 source ~/.bashrc（或 ~/.zshrc，取决于你使用的 Shell）来加载新路径。

然后，运行以下命令来安装 Foundry 工具：

foundryup

验证：安装完成后，检查 forge 命令是否可用。

forge --version

## 第二阶段：安装 ZetaChain CLI 并验证

**1\. 安装 CLI**

目的：这是与 ZetaChain 交互的最主要工具，用于管理账户、发起跨链调用、查询状态等。

操作：

使用 npm 全局安装：

npm install -g @zetachain/zetachain-cli

注意：官方文档的命令是 npm install -g zetachain，但当前更推荐使用 @zetachain/zetachain-cli 包。

**2\. 验证 CLI 安装**

操作：运行以下命令，它会列出所有 ZetaChain 连接的链。如果成功显示一个很长的链列表（如 Ethereum, BSC, Polygon 等），说明你的 CLI 安装正确且能访问网络信息。

zetachain query chains list

## 第三阶段：启动并测试 Localnet（推荐）

**1\. 启动 Localnet**

目的：在本地计算机上启动一个完整的、自包含的测试环境，包含 ZetaChain、EVM 链以及（可选）Solana、Sui 和 TON。这是最快、最免费的开发和测试方式。

操作：

确保 Foundry 已安装后，只需一个命令：

zetachain localnet init

注意：首次运行会下载必要的 Docker 镜像，这可能需要一些时间。请确保你的系统已安装并运行了 Docker Desktop。

**2\. 验证 Localnet 运行状态**

操作：启动后，CLI 会输出日志。你可以使用以下命令检查核心服务的状态：

zetachain localnet status

它会显示 ZetaChain 节点、EVM 链节点等是否在正常运行
